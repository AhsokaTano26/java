# Java Web 实验报告

## 一、实验题目
基于 Java Web 的“情绪垃圾桶匿名墙”系统设计与实现

## 二、实验目的
1. 掌握 JSP + Servlet 的 Web 应用开发流程。  
2. 掌握 JDBC 连接 SQLite 数据库并完成基本 CRUD。  
3. 学习会话管理、参数校验与基础安全防护（密码哈希、CSRF、输出转义）。  
4. 完成一个可部署的 WAR 项目并在 Tomcat 上运行。

## 三、实验环境
- JDK 8+
- Maven 3.8+
- Tomcat 9+
- SQLite（通过 `sqlite-jdbc` 驱动访问）

## 四、项目功能概述
本项目实现了一个匿名情绪墙，核心功能如下：
- 用户登录（基于 `users` 表认证）
- 用户注册（用户名唯一）
- 发布纸条（内容 + 情绪标签）
- 查看纸条（时间轴 / 随机流）
- 按情绪标签筛选纸条
- 删除纸条（仅允许删除本人发布内容）

## 五、系统架构与模块划分
项目采用典型三层组织：

1. **Servlet 控制层**（`src/main/java/com/example/servlet`）  
   负责请求分发、参数校验、会话判断、页面跳转与响应。

2. **DAO 数据访问层**（`src/main/java/com/example/dao`）  
   负责与 SQLite 交互，封装用户认证/注册、纸条查询/新增/删除等 SQL。

3. **Model 与工具层**（`model` + `util`）  
   - `StudentRecord`：纸条实体  
   - `DbUtil`：数据库连接  
   - `PasswordUtil`：PBKDF2 密码哈希与校验  
   - `HtmlEscapeUtil`：页面输出转义

前端页面位于 `src/main/webapp`，使用 JSP 渲染与表单交互。

## 六、数据库设计
`sql/schema.sql` 中定义了两张核心表：

1. `users`
- `id`：主键
- `username`：唯一用户名
- `password`：密码哈希（非明文）

2. `mood_notes`
- `id`：主键
- `content`：纸条内容
- `mood`：情绪标签（`calm/happy/sad/angry/anxious/tired`）
- `owner_username`：发布者用户名（外键关联 `users.username`）
- `created_at`：创建时间

并建立了时间排序索引和发布者索引，提升查询性能。

## 七、关键实现说明
### 1. 认证与会话
- 登录成功后将 `username` 写入 Session。
- 未登录访问匿名墙时自动重定向到登录页。

### 2. 密码安全
- 使用 `PBKDF2WithHmacSHA256` 加盐哈希存储密码。
- 校验时采用常量时间比较，降低时序攻击风险。

### 3. 业务校验
- 纸条内容不能为空、长度不超过 280 字。
- 情绪值严格限制在白名单集合内，避免非法参数写入。

### 4. CSRF 防护
- 在 Session 生成并保存 `csrfToken`。
- 对新增/删除操作在 POST 中进行 token 校验。

### 5. 输出安全
- 页面展示统一经过 HTML 转义，避免脚本注入。

### 6. 查询能力
- 支持按时间轴和随机流查看纸条。
- 支持按心情筛选后再按“时间轴/随机流”展示。

## 八、实验结果

```bash
mvn clean package
```

成功生成 `target/java-web-crud.war`，可部署到 Tomcat 运行。  
系统完成了实验目标中的注册、登录、增删查与筛选功能，流程完整可用。
